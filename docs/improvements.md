## 1. 새로운 페이지/기능 후보

### 1.1 통계/리포트 페이지

- **설명**: 주간/월간 활동 통계, 플랫폼별 기여도 차트, 생산성 트렌드 시각화
- **근거**: 데이터는 이미 수집 중이나 집계 뷰가 없음. 히트맵 외에 다양한 관점의 데이터 시각화 필요
- **예상 구성**:
    - 기간별 커밋/이슈/메시지 건수 추이 차트
    - 플랫폼별 활동 비율 파이 차트
    - 요일/시간대별 활동 패턴 분석

### 1.2 팀 대시보드 (어드민)

- **설명**: 팀원별 활동 현황 비교, 전체 업무 진행률 요약
- **근거**: 어드민이 개별 사용자 데이터를 한눈에 볼 방법이 없음
- **예상 구성**:
    - 팀원별 활동 히트맵 비교
    - 전체 태스크 상태 분포
    - 팀원별 업무일지 생성 현황

### 1.3 알림/노티피케이션 센터

- **설명**: 수집 완료, 업무일지 생성 완료, 태스크 상태 변경 등 알림 이력 관리
- **근거**: 현재 토스트만 있고 이력 관리가 없음. 토스트는 일시적이라 놓치기 쉬움
- **예상 구성**:
    - 헤더/사이드바에 알림 아이콘 + 뱃지
    - 알림 드롭다운 또는 전용 패널
    - 읽음/안읽음 상태 관리

### 1.4 GitHub Secret 관리 페이지 (어드민)

- **설명**: Jira/Slack과 동일한 어드민 Secret 관리 페이지
- **근거**: 현재 GitHub PAT는 개인 프로필에서만 관리 가능. Jira/Slack은 어드민 + 사용자 이원화 구조인데 GitHub만 비대칭
- **참고**: 백엔드에 어드민용 GitHub Secret API가 추가되어야 할 수 있음

### 1.5 설정 페이지

- **설명**: 알림 설정, 테마 세부 조정, 기본 날짜 범위 등 통합 설정
- **근거**: 현재 설정이 분산되어 있음 (사이드바 다크모드 토글, 레이아웃 너비 조절 등)

### 1.6 온보딩 가이드

- **설명**: 첫 사용자를 위한 단계별 설정 안내 (연동 -> 수집 -> 일지 확인)
- **근거**: 빈 상태에서 다음 행동 안내가 부족. 플랫폼 연동 없이 수집해도 데이터가 없다는 것을 모름

## 2. 기존 페이지 기능 강화

### 2.1 에러 시 재시도 버튼 (전체 페이지)

- **현황**: `ErrorMessage` 컴포넌트에 `onRetry` prop이 이미 구현되어 있으나 어느 페이지에서도 사용하지 않음
- **개선**: 각 페이지에서 쿼리 훅의 `refetch` 함수를 `ErrorMessage`의 `onRetry`로 전달
- **영향 범위**: `DashboardPage`, `PlatformWorkLogPage`, `WorkBoardPage`, `UserListPage`, `JiraSecretListPage`,
  `SlackSecretListPage`
- **노력도**: 낮음

### 2.2 WorkBoardPage 로딩/에러 상태 추가

- **현황**: `useGetTasksApiV1TasksGet` 결과에서 `isLoading`, `error`를 구조분해하지 않음. 로딩 중에도 빈 상태("등록된 업무가 없습니다")만 표시되어 로딩/에러/빈 상태
  구분 불가
- **개선**: `isLoading` 시 스켈레톤 로더, `error` 시 `ErrorMessage` + 재시도 버튼 표시
- **노력도**: 낮음

### 2.3 빈 상태에서 플랫폼 연동 안내

- **현황**: `PlatformWorkLogPage` 빈 상태에서 "수동 수집을 시작하여..." 안내만 있음. 플랫폼 연동이 안 되어 있으면 수집해도 데이터가 없다는 안내 없음
- **개선**: `useGetUserMeApiV1UsersMeGet` 데이터로 연동 상태 확인 후, 연동 없으면 "먼저 프로필에서 플랫폼을 연동하세요" 메시지 + `/profile` 링크 표시
- **노력도**: 중간

### 2.4 날짜 상태 URL 반영

- **현황**: `DashboardPage`, `PlatformWorkLogPage`에서 `targetDate`가 `useState`로만 관리. URL에 반영되지 않아 북마크/공유 불가
- **개선**: `useSearchParams`로 `?date=2026-03-06` 형태로 URL 반영. `ProfilePage`에서 탭 상태를 `useSearchParams`로 관리하는 패턴이 이미 확립되어 있음
- **노력도**: 낮음

### 2.5 WorkBoardPage 필터/선택 상태 URL 반영

- **현황**: `filter`(상태 필터)와 `selectedTaskId`가 로컬 상태로만 관리. 페이지 이동 후 돌아오면 초기화됨. 태스크 딥링크/공유 불가
- **개선**: `?filter=IN_PROGRESS&taskId=42` 형태로 URL 상태 유지
- **노력도**: 낮음

### 2.6 태스크 검색 기능

- **현황**: 태스크 수가 늘어나도 상태 필터만 가능. 제목/내용/태그 기반 검색 없음
- **개선**: 컨트롤 바에 검색 입력 필드 추가, 클라이언트 사이드 즉시 필터링
- **노력도**: 중간

### 2.7 TaskEditor 저장 완료 피드백

- **현황**: 디바운스 자동 저장 시 "저장 중..." 텍스트는 표시되지만 저장 완료 후 피드백 없음. 실패 시에만 토스트 에러 표시 (비대칭)
- **개선**: 저장 완료 후 "저장됨" 텍스트 또는 체크 아이콘 짧게 표시 (Google Docs 스타일)
- **노력도**: 낮음

### 2.8 PlatformWorkLogPage 뷰 모드 localStorage 저장

- **현황**: `viewMode`('card' | 'timeline')가 `useState`로만 관리. 새로고침 시 카드 뷰로 초기화
- **개선**: `WorkBoardPage`에서 뷰 모드를 `localStorage`에 저장하는 패턴이 이미 있음. 동일 방식 적용
- **노력도**: 낮음

### 2.9 대시보드 위젯 스켈레톤 통일

- **현황**: `PlatformWorkLogPage`는 `WorkLogCardSkeleton` 사용. 대시보드 4개 위젯은 `LoadingSpinner` 사용. 로딩 UI 패턴 불일치, 레이아웃 시프트 발생
- **개선**: 위젯 내부에 콘텐츠 형태를 모방하는 스켈레톤 적용. `Skeleton` 컴포넌트가 이미 존재
- **노력도**: 중간

### 2.10 수동 수집 진행 상태 안내

- **현황**: 수동 수집 시작 후 버튼이 "수집 중..."으로 바뀌지만, 30초 폴링 간격으로 새 데이터가 올 때까지 진행 상태 안내 없음
- **개선**: "수집 시작됨, 최대 수분 소요될 수 있습니다" 안내 배너 표시
- **노력도**: 낮음

### 2.11 마지막 수집 시각 표시

- **현황**: 언제 마지막으로 데이터를 수집했는지 표시 없음
- **개선**: "마지막 수집: 2시간 전" 형태 정보 표시. 사용자가 재수집 필요 여부 판단 가능
- **노력도**: 낮음

### 2.12 태스크 생성 시 현재 필터 상태 반영

- **현황**: 새 태스크 추가 시 항상 `BACKLOG`로 생성. `IN_PROGRESS` 필터로 보는 중에 추가하면 현재 뷰에서 보이지 않음
- **개선**: 현재 선택된 필터의 상태를 새 태스크 기본 상태로 설정
- **노력도**: 낮음

## 3. UX 품질 개선

### 3.1 폼 인라인 유효성 검사

- **현황**: 모든 폼에서 유효성 검사가 HTML `required`와 서버 에러 토스트에만 의존. 필드 수준 에러 메시지 없음
- **개선**: `FormField` 컴포넌트에 `error` prop 추가. `onBlur` 시 인라인 에러 메시지 표시
- **영향 범위**: `LoginPage`, `UserForm`, `JiraSecretForm`, `SlackSecretForm`, GitHub PAT 폼
- **노력도**: 중간

### 3.2 모달 내 에러 인라인 표시

- **현황**: 모달 폼 에러가 모달 뒤편 토스트로 표시되어 사용자가 놓치기 쉬움
- **개선**: 모달 내부 상단 또는 필드 수준에 에러 인라인 표시. 토스트는 전역 성공/실패 알림에만 사용
- **노력도**: 중간

### 3.3 토스트 `warning` 타입 추가

- **현황**: `success`, `error`, `info` 3가지만 지원. `warning-*` 디자인 토큰이 있으나 토스트에서 미지원
- **개선**: `warning` 타입 추가하여 경고성 알림 표현 가능하게 함
- **노력도**: 낮음

### 3.4 키보드 단축키

- **현황**: `TaskEditor`에 `Cmd/Ctrl + Enter` 저장만 구현. 전역 단축키 없음
- **개선**: `N` (새 태스크), `/` (검색), `?` (단축키 도움말) 등 파워 유저용 전역 단축키
- **노력도**: 중간

### 3.5 AdminRoute 권한 없음 페이지

- **현황**: 비관리자가 어드민 URL 접근 시 토스트 + 즉시 리다이렉트. 토스트를 읽기 전에 화면 전환됨
- **개선**: 전용 403 "권한 없음" 페이지 표시, 또는 리다이렉트 전 지연 추가
- **노력도**: 낮음

### 3.6 날짜 변경 시 document.title 업데이트

- **현황**: 날짜 변경해도 브라우저 탭 제목 불변. 여러 탭 비교 시 구분 불가
- **개선**: `useEffect`로 `document.title`을 `대시보드 - 2026-03-06 | Dev Blackbox` 형태로 업데이트
- **노력도**: 낮음

### 3.7 모바일 칸반 뷰 접근성

- **현황**: 칸반 뷰 토글이 `hidden lg:flex`로 데스크톱 전용. 모바일에서 존재를 모름
- **개선**: 가로 스크롤 칸반 또는 "데스크톱에서 이용하세요" 안내 표시
- **노력도**: 높음

### 3.8 콘텐츠 너비 토글 발견성

- **현황**: 우상단에 작고 `text-text-tertiary` 색상으로 눈에 띄지 않음
- **개선**: 첫 방문 시 툴팁 제공 또는 사이드바 설정 영역으로 이동
- **노력도**: 낮음

## 참고: 잘 구현된 패턴 (다른 영역에서 참조)

- **Modal 컴포넌트**: Escape 닫기, 포커스 트랩, 배경 스크롤 잠금, 포커스 복원 모두 구현
- **PlatformWorkLogPage 폴링**: 새 데이터 감지 시 폴링 자동 중단
- **TaskEditor unmount 시 디바운스 플러시**: 미저장 변경 즉시 저장으로 데이터 손실 방지
- **WorkLogCard 복사 버튼**: 복사 후 아이콘 체크로 변경되는 즉각적 피드백
- **Sidebar**: localStorage 저장, 다크 모드 토글, 관리자 전용 섹션 분리 일관 구현
